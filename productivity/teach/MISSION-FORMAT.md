# MISSION.md Format

`MISSION.md` は workspace root に置く。ユーザーがこのテーマを学ぶ理由を短く固定し、何を教えるか・どのリソースを選ぶか・どの練習を作るかの判断基準にする。

## Template

```md
# Mission: {Topic}

## Why
{1-3 文。ユーザーが追っている現実の目標。これを身につけると仕事・生活・制作物がどう変わるか。"X を理解する" のような抽象表現は避け、背後の成果まで掘る。}

## Success looks like
- {観察可能な到達状態}
- {別の到達状態}
- {…}

## Constraints
- {時間、予算、締切、前提知識、学習スタイルなど}

## Out of scope
- {今は追わない隣接トピック。近接発達領域を守るために書く}
```

## Rules

- **1 workspace = 1 mission。** 無関係なテーマを同じ workspace に混ぜない。
- **具体を優先する。** 「健康になる」より「10 月までにハーフマラソンを走る」。「Rust を学ぶ」より「チーム向け Rust CLI を出荷する」。
- **曖昧なら質問する。** bad mission は no mission より危険。すぐ lesson を作らず、目的を interview する。
- **変化したら更新する。** mission が変わったら古いままにしない。必要なら learning record にも残す。
- **短く保つ。** 1 画面を超えるなら compass ではなく plan になっている。
